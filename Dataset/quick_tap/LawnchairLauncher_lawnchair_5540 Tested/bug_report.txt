Home Button Requires Double-Tap to Return to Default Home Page from Other Home Screens

App: LawnchairLauncher/lawnchair
Version: 15.0.0
Issue: https://github.com/LawnchairLauncher/lawnchair/issues/5540
APK: https://github.com/LawnchairLauncher/lawnchair/releases/download/nightly/Lawnchair.Nightly.apk
APK Tag: nightly
Gestures: tap, quick tap, press

Steps to Reproduce:
  1. Set a specific home page as your default home screen in Lawnchair settings
  2. Swipe or navigate to a different home page (one that is not your default)
  3. Press the home button on your device
  4. Observe that the launcher does not return to your default home page
  5. Press the home button again (double-tap total - quickly)
  6. Observe that the launcher now returns to your default home page

Expected Behavior:
  When on any home page other than the designated default home page, a single press of the home button should immediately return the user to the default home page.

Actual Behavior:
  A single press of the home button does not return to the default home screen. Instead, a double-tap of the home button is required. This occurs consistently regardless of the number of home pages configured.

--- Full Issue Body ---
This issue occurs consistently regardless of the number of home pages I have configured. For some reason, it just "refuses" to go to the home page if just pressed the Home Button once.
