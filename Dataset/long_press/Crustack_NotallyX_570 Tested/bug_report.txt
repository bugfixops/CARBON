### What happened?

**Bug Description**
The app crashes when long-pressing to select a note after filtering the note list if the previously selected note is no longer visible due to a search.

**To Reproduce**
Steps to reproduce the behavior:

1. Add at least two notes.
2. Long-press to select one note (Note A).
3. Use the search function so that Note A disappears from the filtered list.
4. Long-press on another visible note in the filtered list.

**Expected Behavior**
Long-pressing a note should always succeed, even if previous selections are invalidated by list filtering. No crash should occur.
